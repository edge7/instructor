# Claude Code Integration

> **Status**: Experimental
>
> This page explains how to use the *Claude Code* SDK together with the
> `instructor` library to obtain structured JSON outputs. The integration is
> provided by the `ClaudeCodeProvider` helper class that lives in
> `provider_claude.py`.

## Installation

```bash
pip install instructor[claude-code]
# or, if you prefer requirements files
pip install -r requirements.txt  # make sure it contains `claude-code-sdk`
```

Requirements:

* Python 3.10+
* Node.js (needed for the Claude Code CLI)
* Claude Code CLI:
  ```bash
  npm install -g @anthropic-ai/claude-code
  ```

## Quick start

```python
import instructor
from pydantic import BaseModel
from provider_claude import ClaudeCodeProvider

class Haiku(BaseModel):
    title: str
    poem: str
    author: str

# ➊ Initialise the provider
claude = ClaudeCodeProvider()

# ➋ Ask Claude to generate a *strict* JSON response
result: Haiku = claude.create(
    prompt="Write a haiku about foo.py",
    response_model=Haiku,
)
print(result)
```

Behind the scenes the provider will:

1. Convert the chat messages into a single prompt string.
2. Append an instruction telling Claude to respond **only** with JSON that
   matches the schema of your `response_model`.
3. Stream the response, extract the JSON payload and validate it with
   Pydantic. You get a fully-typed `Haiku` object back.

## Using with `instructor`

You can still take advantage of the broader *instructor* feature-set (hooks,
validation retries, etc.) by calling the provider directly inside your custom
logic, or patching it into your own `Instructor` subclass.

## Limitations

* The provider currently flattens messages into a single prompt; multi-turn
  conversations are not yet supported.
* Ensure that both the Python SDK **and** the CLI are installed.
